## 引言：每个文件夹系统的隐藏税

每次你创建一条笔记，你都要交税。

你把想法写下来。然后你停下来。这是项目吗？资源吗？领域？它实际属于哪个文件夹？你花三十秒决定。你选了一个。然后继续。在你脑海深处，你知道选错了，或者至少可以选不同的。

每次的税很小。但对于任何认真做知识工作的人来说，每周会发生数百次。还有第二次税，在检索时支付：你不仅需要记住你写了什么，还要记住你放在哪里了。你打开文件夹，扫描子文件夹，然后空手而归。

本应释放你思维的系统反而在消耗它。

我在那个循环里待了多年。我试过 PARA。我试过卡片盒。我试过把两者混合。系统很聪明，但它们都基于同一个有缺陷的假设：笔记属于位置。我终于在停止思考笔记去哪里、开始思考笔记是什么的时候找到了出路。

## 为什么 PARA 永远不会奏效

我之前的系统是一个我叫做 IPARAG 的混合体：收件箱、项目、领域、资源、档案和银河。PARA 部分来自 Tiago Forte，处理项目和资源管理。银河是我的卡片盒层，我在这里以扁平文件结构保存所有永久笔记，通过维基链接连接。

银河真的很强大。一旦我有足够多的原子笔记互相链接，内容创作变得很快。我可以从自己的思考中提取，而不是每次从零开始。那部分有效。

其余的有结构缺陷，再多改进也无法修复。

缺陷是这样的：一条笔记可以同时属于多个位置。关于内容策略的笔记是项目文件还是资源？它在商业领域还是内容文件夹？PARA 没有回答这个问题。它迫使你选一个，然后带着疑虑生活。每次你检索东西时，你都要猜测过去的你是否正确归档了。

系统也是为基于文件夹的工具构建的。当时我用 Kortex 和 Eden 管理我的第二大脑。在基于文件夹的世界里，PARA 可能是最好的解决方案。但我现在使用的工具是 Obsidian，而 Obsidian 最近发布了一个改变一切的功能：Bases。

我花了一周学习如何使用它，主要是研究 Kepano 如何使用它。Kepano 是 Obsidian 的创始人。一周后，我从零开始重建了整个保险库。这就是我要在这里带你了解的结果。

## 架构：五个文件夹，无层级

整个系统运行在五个文件夹上。

收件箱是捕获区。所有未处理的东西都落在这里。随机的想法、下载的文件、语音备忘录、网络剪藏。你不把任何东西分类到收件箱。你只是把它扔进去，之后再处理。Claude Code 自动处理我的，但我稍后会讲到。

笔记文件夹是系统中每一条笔记所在的地方。一个扁平文件夹，没有子文件夹，没有例外。九百个文件还在增加，都在同一级别。这不是混乱。这是刻意的，它有效的原因是下两个文件夹。

类别和主题是导航层。它们不是存储。笔记不住在那里。这些文件夹包含容器笔记，动态地从扁平笔记文件夹中浮现和组织笔记。其工作原理的细节是系统的核心。

这个架构代表的核心哲学转变很简单：文件夹系统要求你在存储之前决定笔记属于哪里。这个系统只问一个问题：这条笔记是什么？一旦你回答了，系统就会为你找到它。

## 类别如何工作：过滤器，不是文件夹

类别是你保险库中存在的一种对象类型。永久笔记、时事通讯、YouTube 视频、推文、SOP、评论、AI 提示、项目、产品。每条笔记至少属于一个类别。

与文件夹的关键区别：一条笔记可以同时属于多个类别。YouTube 脚本可以既是 YouTube 视频又是脚本。时事通讯可以既是时事通讯又是素材库。笔记只存在一次，在扁平的笔记文件夹里。类别只是描述它是什么。

为了实现这一点，每个类别都是一个带有嵌入 [Obsidian Bases](https://obsidian.md/) 视图的容器笔记。Bases 是内置于 Obsidian 的核心插件。你不需要下载任何东西。它把你的笔记变成数据库，类似于 Notion 所做的，但使用你本地机器上的纯 markdown 文件。

每个类别容器内的数据库视图有一条规则：显示这个保险库中链接到该类别的每一条笔记。

链接通过 YAML frontmatter 工作，这是每条笔记顶部的一组元数据。一份时事通讯的元数据中有这个：categories: "[[Newsletters]]"。这一行就是到时事通讯容器的维基链接。一旦你添加它，笔记就会出现在时事通讯容器中，无需任何额外操作。不需要分类、不需要拖动、不需要归档。

这是 PARA 永远无法实现的。笔记不被困在一个位置。它可以从你想到查找的任何角度被找到。

## 主题如何工作：第二个导航层

类别回答一个问题：这是什么类型的笔记？主题回答另一个问题：这是关于什么的？

主题是跨所有类别的主题。商业、创意、心理学、哲学、健康、生产力、人际关系等。任何笔记都可以在类别之外添加主题。关于心理学的时事通讯有两个属性：categories: [[Newsletters]] 和 subjects: [[Psychology]]。

那份时事通讯出现在时事通讯容器中，与其他所有时事通讯并列。它也出现在心理学主题容器中，与保险库中所有其他心理学笔记并列，包括永久笔记、推文、视频和任何带有该主题属性的其他内容。同一条笔记，从两个完全不同的角度浮现，只需要在元数据中添加一项。

这取代了我之前使用的整个标签系统。Obsidian 中的标签在技术上是有效的，但它们是不可见的。你忘记存在哪些标签了。它们堆积起来。你没有办法轻松浏览或探索它们。主题容器是可见的、有名称的、可导航的。你像打开页面一样打开它们。这是一种更清晰、更强大的知识库导航方式。

## 捕获如何工作：一个动作，完全组织好

日常捕获工作流程很简单。

你创建一条笔记。它自动落入扁平的笔记文件夹。你应用匹配的模板。时事通讯、推文、永久笔记、YouTube 视频，每个类别都有自己的模板。模板自动填充 YAML frontmatter：类别链接、状态、类型。你不需要自己写任何这些。你只需要应用模板并开始写作。

一旦你应用模板，笔记就出现在正确的类别容器中。系统已经组织好了。

模板不过是一个预先写好的 YAML 块加上内容结构。当你应用时事通讯模板时，它添加类别链接并将状态设置为"想法"。你得到一份空白文档准备填写，预路由到正确位置。

当我意识到我已经完全停止思考把笔记放在哪里时，我豁然开朗了。我只思考我在写什么。这个转变听起来很小。但不是。捕获阶段的摩擦是杀死第二大脑系统的原因。大多数人构建了某个系统，用了几周，然后放弃了，因为每次捕获都需要做决定。去掉那个决定，你才真正使用这个系统。

## 当 Claude Code 住在你的保险库里时会发生什么变化

Claude Code 是一个直接在 Obsidian 文件夹内运行的 AI agent。它读取你的文件、写入文件，并在每个会话的第一个提示就理解你的系统。

它总是理解你的系统的原因是 CLAUDE.md 文件。保险库中的每个文件夹都有一个。这些文件解释那个文件夹里有什么、类别意味着什么，以及笔记应该如何组织。Claude 在每个会话开始时自动读取它们。你配置一次。Claude 每次都捡起来。

实际结果是让你永远不需要考虑的维护层。

你可以让它创建关于任何主题的时事通讯笔记，Claude 会用正确的 YAML、正确的模板和正确的状态创建文件。你可以 说"处理我的收件箱"，Claude 读取收件箱文件夹中的每个文件，分配正确的类别和主题，然后移动到笔记。你可以 说"给每条提到心理模型的时事通讯添加心理学主题"，Claude 在二十秒内更新三十条笔记。

这个组合是精确的：Obsidian Bases 从视觉上组织你的知识。Claude Code 自动维护它。你使用系统。Claude 运行它。自从我设置了这个，我没有手动管理过我的数据库。

这解锁的另一个维度是内容创作。因为 Claude 可以访问你整个第二大脑，你曾经写过的每一条笔记、你曾经创作过的每一条内容，它都用那个作为上下文和源材料。例如，你可以让 Claude 写一份时事通讯，它会综合你保险库中五条相关笔记，用你的声音写作，并使用你自己的想法作为源材料。使用通用提示的通用 AI 产生通用输出。拥有组织良好的知识库完整上下文的 AI 产生完全不同的东西。
